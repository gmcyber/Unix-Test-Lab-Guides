Network Design

Firewall

IP Address: 192.168.100.1/24

Hostname: firewall.college.com

The Firewall’s purpose is to handle traffic between the Intranet and Internet. This means that it will act as a router directing traffic. There are two interfaces on Firewall, one is the external/Internet-facing called ens192 and the other is the internal/Intranet-facing called ens224.

DC1

IP Address: 192.168.100.2

Gateway: 192.168.100.1

DNS Server: 127.0.0.1

Hostname: dc1.college.com

This will be configured as the Intranet’s Domain Name System (DNS) and the Dynamic Host Configuration Protocol (DHCP) server.

App1

IP Address: 192.168.100.3

Gateway: 192.168.100.1

DNS Server: 192.168.100.2

Hostname: app1.college.com

This is an internal web and application server.

Client1 IP Address: Dynamic

DNS Server: 192.168.100.2

Hostname: client1.college.com

Client1 is a workstation that has the ability to switch between the Intranet and Internet subnet.

Configure TCP/IP

  1. Ensure that Network Manager is running. Type systemctl status NetworkManager.

    1. You should see “Active: active (running)” in green if it is running.

    2. If it is not running, enter the command sudo systemctl start NetworkManager.service.

    3. If the service fails to launch, check the logs by entering the command sudo systemctl status NetworkManager.service -l.

  2. This is for the Internet-facing interface. Enter the command nmtui to enter the Network Manager.

    1. Select “Edit a connection.”

    2. Select the Internet-facing interface, ens192, and use the arrow keys to select “<Edit…>”

    3. Use the arrow keys to ensure that “IPv4 CONFIGURATION” is set to “<Automatic>”

    4. At the very bottom, ensure that the brackets in front of “Automatically connect” have an X by hitting the spacebar while highlighting them.

    5. Use the arrow keys to select “<OK>” then “<Back>” and press Enter to go back to the main menu of Network Manager.

    6. Use the arrow keys to select “Quit” and press Enter to exit the editor.

  3. Now, repeat for Intranet-facing interface.

    1. Select “Edit a connection.”

    2. Select the Intranet-facing interface, ens224, and use the arrow keys to select “<Edit…>”

    3. Use the arrow keys to ensure that “IPv4 CONFIGURATION” is set to “<Manual>”

    4. Under “Addresses” add in “192.168.100.1/24”

    5. At the very bottom, ensure that the brackets in front of “Automatically connect” have an X by hitting the spacebar while highlighting them.

    6. Use the arrow keys to select “<OK>” then “<Back>” and press Enter to go back to the main menu of Network Manager.

    7. Use the arrow keys to select “Quit” and press Enter to exit the editor.

  4. Change the hostname. To change the hostname to firewall.college.com use the following command: sudo hostnamectl set-hostname firewall.college.com.

  5. Restart the network using sudo systemctl restart network.

  1. Run the command sudo yum install nano -y.

Configure NAT Rules

This will allow the system to act as a router and to allow traffic from the internal network out to the external network.

  1. Enable IP forwarding by entering the command sudo echo “net.ipv4.ip_forward = 1” >> /etc/sysctl.d/ip_forward.conf.

    1. You will need to be root to perform this task.

    2. Switch back to champlain user afterwards using su champlain.

  2. Add the external-facing NIC to Firewalld’s “public” zone by typing: sudo firewall-cmd --zone=public --add-interface=ens192 --permanent

    1. Where ens192 is the name of the external interface

  3. Add the internal-facing NIC to Firewalld’s “internal” zone by typing: sudo firewall-cmd --zone=internal --add-interface=ens224 --permanent

    1. Where “ens224” is the name of the internal interface

  4. Apply masquerading to the external Firewalld zone by typing: sudo firewall-cmd --zone=public --add-masquerade --permanent

  5. Save those changes by typing: sudo firewall-cmd --complete-reload

  6. Enable NAT by entering the command sudo firewall-cmd --permanent --direct --passthrough ipv4 -t nat -l POSTROUTING -o ens224 -j MASQUERADE -s 192.168.100.0/24.

  7. Restart the firewall to save using sudo firewall-cmd --reload.

  8. Configure NAT rules on iptables. Enter the following commands. sudo iptables -F sudo iptables -P INPUT ACCEPT sudo iptables -P OUTPUT ACCEPT

  9. Ensure NAT rules on iptables remains on boot. Add in the commands from earlier for NAT by entering the command sudo nano /etc/rc.local. This will open the file in Nano Editor.

    1. Add in the following at the end:

      # NAT Rules
iptables -P INPUT ACCEPT
iptables -P OUTPUT ACCEPT

  1. The above ensures that the NAT rules are still in place on reboot.

  2. Press Ctrl+X to exit the Nano Editor then hit Y to save the file and hit Enter to save it with the same name.

Congratulations, Firewall is now configured.

Troubleshooting

Location to Logs & Configuration Files

  • General Logs: /var/log/messages

  • Samba Configuration: /etc/samba/smb.conf

I. Firewall - Installing Nano - Cannot install nano

  1. Check to see if you can ping out by using ping 8.8.8.8. If you can’t then this should be a solution.

  2. Sometimes, when interfaces are edited in Network Manager, there will be a second ens192. Delete it.

  3. First check Network Manager by using nmtui to see if the interfaces are set up properly.

    1. The external/Internet interface, ens192, should have:

      1. IPv4 Configuration set to Automatic

      2. Automatically connect check-boxed

    2. The internal/Intranet interface, ens224, should have:

      1. IPv4 Configuration set to Manual

      2. IP Address: 192.168.100.1/24

      3. Automatically connect check-boxed

  4. Restart your network by using sudo systemctl restart network.

II. Firewall - systemctl restart network

  1. It’s okay if network won’t restart.

III. Firewall - tcpdump

  1. Install tcpdump on your Firewall: sudo yum install tcpdump -y

  2. Switches:

    1. -D: Display available interfaces

      • Ex: sudo tcpdump -D

    2. -i: Capture packets from a specific interface

      • Ex: sudo tcpdump -i ens224

    3. -n: Capture IP address

      • Ex: sudo tcpdump -n

    4. port: Captures from a specific port

      • Ex: tcpdump -i ens224 port 22

    5. src: Captures packets from source IP

      • Ex: tcpdump -i ens224 src 192.168.0.2

    6. tcp: Captures only TCP packets

      • Ex: sudo tcpdump -i ens224 tcp
